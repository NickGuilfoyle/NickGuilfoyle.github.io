EECS 183 Elementary Programming Concepts

  • Project 1 Shopping List

    Implemented program to calculate the cost of a shopping list in C++, and to print out the list along with the prices of items. First CS project I completed at Michigan.

  • Project 2 Rock Paper Scissors

    Implemented the game rock paper scissors in C++. Project served as the introduction to loops, and if statements.

  • Project 3 OhHi

    Wrote the functions for the rules to OhHi in C++. OhHi is a game similar to Sudoku, but less complicated. Project was a further exploration of conditionals, and was also an introduction to writing our own functions.

  • Project 4 Bitmap

    Wrote some functions in a .bmp editing program in C++, which gave us the ability to manipulate and display .bmp images. Project was the introduction to the manipulation of arrays

  • Final Project: Creative AI

    Much larger than the previous four projects, Creative AI was a basic machine learning agent which used weighted probability learning to write Beatles songs using a dictionary of the band's music. The project was then expanded upon by my group to include a twitter bot using tweepy to emulate accounts on twitter using TweePy. Project served as an introduction to Python, an introduction to using code from the web, and a basic introduction to machine learning.

EECS 280 Programming and Introductory Data Structures

  • Project 1 Stats

    Serving as the refresher to C++, this was a short project which required students to write functions to compute summary statistics given a dataset, by processing data into the appropriate structures, then sending the structures into functions to compute mean, median, mode, etc. This project did not necessarily introduce any new concpets, but served as a comprehensive review of the introductory course.

  • Project 2 Computer Vision

    Project 2 involved an implementation of a basic image compressor in C++. The Project was centered around getting used to more complicated data structures than had been used in the introductory course, specifically manipulation of 2-dimensional vectors. Project also was an introduction to classes.

  • Project 3 Euchre

    Project was to implement the card game Euchre in C++. Much longer than the projects before it, this project involved filling out classes to make functional card decks, implementing simple AI players, and writing a driver from scratch for the first time. Project was a further exploration of classes, and introduction to inherited classes, and an introduction to creating program files without instructor code.

  • Project 4 Reverse Polish Notation Calculator

    Project 4 was a multi-part project where stack and queue introduced in C++, then the student modified stack and queue classes were used to implement a reverse polish notation calculator. Project served as an introduction to stack and queue.

  • Project 5 Machine Learning

    Project 5 was another multi-part project, this time with binary search tree and map being the data structures partially written by students in C++. Then, these structures were used to implement a log-likelihood machine learning application to classify help forum posts. This project was an introduction to tree and map structures, the first time that students needed to work with pointers in many places, and the machine learning application required students to make their own data structures to carry out computations.

EECS 281 Data Structures and Algorithms

  • Project 1 Maze Solver

    Maze Solver was the first project I wrote at Michigan where I had to structure the entire project on my own. The project implemented breasummaryh-first and depth-first search algorithms in C++ using queue and stack. These algorithms solved mazes comprised of characters, and were designed with efficiency in mind, being tested on speed for maze files as large as 10,000 rows. Project was an introduction to coding for efficiency, and for structuring a project without skeleton code from professors.

  • Project 2 Star Wars

    Mult-part project centered on the concept of priority queues. The Star Wars application was the largest part of the project where text about Star Wars Battalions was processed into Jedi and Sith priority queues, then processed to create stories of fictional battles. THe project also featured implentations of a basic priority queue, a binary heap, and a pairing heap. Project was mainly to serve as an introduction to priority queues and heaps, and was also our first major implementation of a data structure.

  • Project 3 Logman

    Logman was a program which centered around using maps and vectors to efficiently create search results given a list of error logs in C++. Logs could be sorted or searched based on multiple attributes of multiple types. Project was the first time that I worked with search and sort algorithms in the context of efficiency, and the project was a much deeper exploration of maps and sets than any previous project.

  • Project 4 Travelling Salesman

    Project 4 was a three part project, with every part relating to the travelling salesman problem, and being coded in C++. Part one was a minimum spanning tree implemented using Kruskal's algorithim. Part 2 used the MST's from part 1 to quickly approximate the true value for the travelling salesman problem for large datasets using branch and bound. Part 3 was an exhaustive search to find the true value of travelling salesman on small datasets. This project was an introduction to Branch and Bound, and part 2 required student's to create an algorithm to quickly approximate travelling salesman on their own.

EECS 445 Introduction to Machine Learning

  • Project 1 Perceptron Algorithm

    For project 1 in EECS 445 we implemented perceptron's algorithim in Python using numpy and scikit to classify IMDB reviews of movies. This project was an introduction to Perceptron's algorithim for binary classification, as well as an introduction to the types of data processing requried for Machine Learning application.

  • Project 2 Convolutional Neural Network

    Project 2 in EECS 445 was an image compressor and decompressor using a Convolutional Neural Network to compres and decompress movie posters with minimal detail loss. The project was completed using python, with numpy and scikit learn. The project was an excellent application of Neural Networks, and provided some insight into how machine learning algorithms can be used for image processing.

EECS 492 Introduction to Artificial Intelligence

  • Sudoku

    In the Sudoku project in EECS 492 students implemented both a backtracking method, and constraint satisfaction method to solve sudoku puzzles in python. The constraint satisfaction model was updated as the course progressed, and by the end of the course solving puzzles with very few iterations using the AC-3 domain constraint algorithms.

  • Treasure Map

    A recurring project where a treasure ship navigated a map to get treasure and avoid hazards in python. The project focused on utility, starting as supervised learning through a markov decision process, and leading to unsupervised learning with nearly the same expected utility. The project utilized Q-Learning to learn a high utility map without prior knowledge of game rules.

  • Pac Man

    Pac Man is hungry was a project which expanded upon breasummaryh first and depth first search, using priority searches, and iterative deepening searches to get pac man through a maze in as few steps as possible.

These queries were an exploration into queries with mutliple joins having multiple conditions, and it was the first time I had written in Javascript.
